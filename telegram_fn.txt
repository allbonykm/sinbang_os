텔레그램 봇 알림 기능 구현 절차 가이드 (신방 OS 적용용)

본 문서는 올본 OS에서 구현된 텔레그램 알림 시스템을 신방 OS에도 동일하게 구축하기 위한 단계별 기획서입니다.

1. 텔레그램 봇 준비 및 정보 획득 (상세 가이드)

   가. 봇 생성하기 (@BotFather 활용)
      - 텔레그램 검색창에 '@BotFather'를 검색하여 공식 인증 마크(파란색 체크)가 있는 봇을 선택합니다.
      - 하단의 [시작(Start)] 버튼을 누릅니다.
      - 채팅창에 '/newbot' 명령어를 입력합니다.
      - 봇의 이름(Name)을 설정합니다. (예: 신방알리미)
      - 봇의 사용자명(Username)을 설정합니다. 반드시 'bot'으로 끝나야 하며 중복될 수 없습니다. (예: sinbang_alert_bot)
      - 성공하면 "Done! Congratulations on your new bot." 메시지와 함께 'API Token'이 발송됩니다.
      - 중요: 이 토큰(예: 8499685968:AAH2uJ3Z...)은 외부로 유출되지 않도록 주의하십시오.

   나. 봇 활성화 (매우 중요)
      - 생성된 봇의 사용자명 링크(예: t.me/sinbang_alert_bot)를 클릭하여 봇 채팅방으로 이동합니다.
      - 하단의 [시작(Start)] 버튼을 반드시 눌러야 합니다. (봇이 사용자에게 먼저 메시지를 보낼 권한을 얻는 과정입니다.)

   다. Chat ID(수신자 ID) 확인하기
      - 알림을 받을 사람 또는 그룹의 고유 식별 번호가 필요합니다.
      - 개인 채팅 ID 확인:
        * 텔레그램 검색창에 '@userinfobot'을 검색하여 시작합니다.
        * 내 ID(숫자 10자리 내외)가 즉시 표시됩니다.
      - 그룹 채팅 ID 확인:
        * 알림을 받을 그룹방을 만들고 생성한 봇을 멤버로 추가합니다.
        * 브라우저 주소창에 다음 주소를 입력하여 접속합니다 (TOKEN 자리에 내 봇 토큰 입력).
          https://api.telegram.org/bot[TOKEN]/getUpdates
        * 브라우저에 표시되는 JSON 내용 중 "chat":{"id": -123456789...} 부분을 찾습니다. 그룹 ID는 보통 마이너스(-)로 시작합니다.

2. 환경 설정 (.env 파일)
   - 프로젝트 루트의 .env 파일에 다음 변수들을 추가합니다.
     TELEGRAM_BOT_TOKEN=획득한_토큰_값
     TELEGRAM_CHAT_ID=획득한_채팅_ID
     TELEGRAM_NOTIFICATIONS_ENABLED=true

3. 텔레그램 전송 유틸리티 작성 (utils/telegram.js)
   - 공통으로 사용할 전송 함수를 정의합니다.
   - 필수 라이브러리: dotenv (환경변수 로드용)
   - 핵심 로직:
     * https://api.telegram.org/bot[TOKEN]/sendMessage 주소로 POST 요청
     * 파라미터: chat_id, text, parse_mode (HTML 추천)
     * 예외 처리: 전송 실패 시에도 메인 시스템이 멈추지 않도록 try-catch 적용

4. 시스템 통합 및 호출
   - 알림이 필요한 모듈(예: 예약 동기화, 에러 핸들러, 비급여 등록 등) 상단에서 유틸리티 임포트
     const telegram = require('../utils/telegram');
   - 특정 이벤트 발생 시 메시지 구성 및 전송 호출
     const msg = `<b>[알림 제목]</b>\n상세 내용...`;
     telegram.sendTelegramAlert(msg);

5. 활용 시나리오 (예시)
   - 비급여/처방 등록 시 실시간 매출 현황 보고
   - 자동 예약 동기화 완료 후 성공/실패 통계 보고
   - 서버 치명적 오류(DB 연결 끊김 등) 발생 시 긴급 알림
   - 환자 만족도 설문 접수 시 즉시 공유

6. 주의 사항
   - 메시지 내 특수문자(<, >, & 등)가 포함될 경우 HTML parse_mode에서 오류가 날 수 있으므로 적절한 이스케이프 처리가 필요할 수 있습니다.
   - 비동기 호출(Promise)을 사용하되, 전송 결과를 기다리느라 사용자 응답이 늦어지지 않도록 주의합니다.
